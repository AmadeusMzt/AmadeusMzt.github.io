学习java的第一天（java开坑）

杨泽华 · 2020-2-26 · 次阅读

俗话说万事开头难，今天刚接触java，首先最重要的是配置环境，学计算机的要学会自己尝试配置环境，我自己琢磨了琢磨写了这样一个教程和心得，欢迎大家指出错误。

初入java

java的背景


兄弟，来喝杯咖啡吗（雾···
Java是一门面向对象编程语言，不仅吸收了C++语言的各种优点，还摒弃了C++里难以理解的多继承、指针等概念，因此Java语言具有功能强大和简单易用两个特征。Java语言作为静态面向对象编程语言的代表，极好地实现了面向对象理论，允许程序员以优雅的思维方式进行复杂的编程。
Java具有简单性、面向对象、分布式、健壮性、安全性、平台独立与可移植性、多线程、动态性等特点。Java可以编写桌面应用程序、Web应用程序、分布式系统和嵌入式系统应用程序等。
上面是百度百科的概述。

总的来说，java是基于c++的编程思想，取其精华，去其糟粕，而产生的一门面向对象编程语言。

java语言可以使用很多当然，也是目前最火的语言，没有之一。是我们做项目，做软件等等很好的工具。

安装以及环境配置

文件下载

首先下载JDK
JDK是 Java 语言的软件开发工具包，主要用于移动设备、嵌入式设备上的java应用程序。JDK是整个java开发的核心，它包含了JAVA的运行环境（JVM+Java系统类库）和JAVA工具。
点此下载
然后安装，这个应该没有问题。
安装到哪个盘都可以，但是一定要记得路径，用来配置环境变量path和classpath。

但是这里有一个问题

学过c和c++语言，初入Java的同学会问：安装好了，咋没有编译器啊？那我要如何运行java？

这和我们学c++的时候有很大区别，当时是先让我们安装集成开发环境（IDE）。大部分同学用的是VC++，少部分用的devc++。这些都是编译器，

但是有没有想过程序是怎么通过编译器运行的？这涉及很多操作系统的知识，然而学习java的原理就需要学习这方面知识。今天我们先配置环境，这部分内容又要开一个博文。

编译工具

我列举其中几种：
Eclipse：一个开放源代码的、基于Java的可扩展开发平台。
NetBeans：开放源码的Java集成开发环境，适用于各种客户机和Web应用。
IntelliJ IDEA：在代码自动提示、代码分析等方面的具有很好的功能。
MyEclipse：由Genuitec公司开发的一款商业化软件，是应用比较广泛的Java应用程序集成开发环境。
EditPlus：如果正确配置Java的编译器“Javac”以及解释器“Java”后，可直接使用EditPlus编译执行Java程序。

然而博主我用的是Vscode，个人感觉这个文本编写工具可以说是特别轻便，而且可以安装很多有用的插件，敲代码手感挺不错的。

下载地址的话，百度即可找到。

配置环境

配置path

桌面找到此电脑，然后右键选择属性，弹出一个窗口，点击右上角的高级系统设置。

会弹出下图的窗口，点击环境变量。

这里要配置环境变量path，找到以后双击。

新建一个环境变量，然后点击浏览，找到你安装JDK的位置，记住字文件夹中一定要包含java.exe，一般来讲是在bin文件夹中。

点击确定，然后将输入的路径通过上移按钮移动到Oracle的上面，这点很重要。

这样path环境变量就配置好了。

验证是否配置成功

点击左下角搜索，输入cmd

打开命令提示符，输入java。如果看到是下图的情况，就代表path配置完毕。

编写我们的第一个java应用程序

那还用说吗？当然是Hello World！了

1.我们随便找个地方创建一个扩展名为.java的文件。
这里，我创建的是HelloWorld.java。

2.然后就是我们的代码环节了，博主刚入坑不久，java的语法只会一部分，但是不管学什么语言，一定要自己写，不能忘了英语单词就去复制。

public class HelloWorld{
    public static void main(String args [])
    {
        System.out.println("Hello World"); //java的输出类比c语言printf()
    }
}

解释一下，看到public class，就想到了c++中的类，没错，java和c++都是面向对象的编程语言。因为java是基于c++的思想的，所以我们在学习的时候应该学会对比学习。
然而 

 public static void main(String args [])

这是什么呢？这是java程序的入口地址,java虚拟机运行程序的时候首先找的就是main方法。跟C语言里面的main()函数的作用是一样的.只有有main()方法的java程序才能够被java虚拟机欲行,可理解为规定的格式。

对于里面的参数及修饰符，
public:表示的这个程序的访问权限,表示的是任何的场合可以被引用,这样java虚拟机就可以找到main()方法,从而来运行javac程序。

static:表明方法是静态的,不依赖类的对象的,是属于类的,在类加载的时候main()方法也随着加载到内存中去

void:main()方法是不需要返回值的
main:约定俗成,规定的
String[] args:从控制台接收参数

3.下一步就是打开cmd，在里面编译并运行刚才写的java程序（博主小时候玩电脑经常在cmd里乱按，没想到现在对此操作如此熟练23333）
首先要找到路径，如下图所示：

先输入cd\，目的是退回根目录，然后输入e:是转到e盘的路径，接着再cd一下我放HelloWorld.java的路径
接下来就是编译和运行了。后面先输入javac HelloWorld.java进行编译，这时候你会发现，在你的HelloWorld.java旁边会出现一个HelloWorld.class这个文件。
之后再输入java HelloWorld.java


这样我们第一个程序就完成了。

配置环境变量classpath

就是类的路径，配置好以后，打开cmd直接输入java HelloWorld即可运行。
还是刚才的步骤打开环境变量的设置，自行添加一个变量，变量值为,;+你的class的路径。

配置好以后，打开cmd，直接输入java HelloWorld即可

使用编译器

博主这里用的是vscode

这样写起来就方便多了。

这里发现一个小问题

我又写了一个switch语句，里面涉及到了中文。

public class Test {
    public static void main(String args[]){
       //char grade = args[0].charAt(0);
       char grade = 'A';

       switch(grade)
       {
          case 'A' :
             System.out.println("优秀"); 
             break;
          case 'B' :
          case 'C' :
             System.out.println("良好");
             break;
          case 'D' :
             System.out.println("及格");
             break;
          case 'F' :
             System.out.println("你需要再努力努力");
             break;
          default :
             System.out.println("未知等级");
       }
       System.out.println("你的等级是 " + grade);
    }
 }

在vscode上可以正常编译并运行。

但是用cmd的话却是乱码，这是怎么回事？

我在右下角发现了，vscode用的是UTF-8字符码编译的，然而用cmd运行的不是，所以要统一一下。
就输入了javac -encoding UTF-8 Test.java就可以编译了！
